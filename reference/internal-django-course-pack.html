The IMDb Data Model

All assignments share these five entities. Design these models first (Assignment 1, Task 1).

EntityFieldsRelationships
Movie name (≤100), movie_id (≤100, unique), release_date, box_office_collection_in_crores (float) One Director (required); many Actors via Cast
Actor actor_id (≤100, unique), name (≤100) Many Movies via Cast
Director name (unique) Directs many Movies
Cast role (≤50), is_debut_movie (bool, default False) Links a Movie ↔ Actor (an actor may have multiple roles)
Rating rating_one_count … rating_five_count (each int, default 0) One per Movie
Modelling hint
Cast is an explicit through-model for the Movie↔Actor many-to-many (it carries role and is_debut_movie). Prefer reverse relations / related fields over multiple manual queries.

Assignment 1 — IMDb

source: django_assignment_002

Build the schema, populate it, and perform basic ORM operations.

  1. Write the models. Define Movie, Actor, Director, Cast, Rating with the fields and relationships in the table above.

  2. Populate the database from the given dictionaries (actors, movies, directors, ratings).

    populate_database(actors_list, movies_list, directors_list, movie_rating_list)

  3. Count the distinct number of movies a given actor acted in.

    get_no_of_distinct_movies_actor_acted(actor_id)
    Returns: integer (e.g. 4)

  4. Return all movies directed by a given director.

    get_movies_directed_by_director(director_obj)
    Returns: list of Movie objects

  5. Return the average rating of a movie (return 0 if it has no rating).

    get_average_rating_of_movie(movie_obj)
    Returns: float (e.g. 4.5)

  6. Delete the rating object associated with a movie.

    delete_movie_rating(movie_obj)

  7. Get all distinct actors who acted in a given list of movies.

    get_all_actor_objects_acted_in_given_movies(movie_objs)
    Returns: list of Actor objects

  8. Update the director for a given movie.

    update_director_for_given_movie(movie_obj, director_obj)

  9. Get all distinct movies that have an actor whose name contains "john".

    get_distinct_movies_acted_by_actor_whose_name_contains_john()

  10. Remove all actors from a given movie — without deleting the actor rows.

    remove_all_actors_from_given_movie(movie_obj)

  11. Get all rating objects for a list of movies (ignore movies with no rating).

    get_all_rating_objects_for_given_movies(movie_objs)

Assignment 2 — IMDb Contd-1

source: django_assignment_003

Reuse the same models. Practice field lookups, date filters, and complex conditions.

  1. Return complete movie info (cast & crew, box office, release) for a list of movie titles.

    get_movies_by_given_movie_names(movie_names)

  2. Movies released in May, June & July, between 2005 and 2010.

    get_movies_released_in_summer_in_given_years()

  3. Movie names having at least one actor whose name ends with "smith" (case-insensitive).

    get_movie_names_with_actor_name_ending_with_smith()

  4. Movie names with at least 1000 and at most 3000 five-star ratings.

    get_movie_names_with_ratings_in_given_range()

  5. 21st-century movies (year > 2000) with ≥ 500 five-star OR ≥ 1000 four-star ratings.

    get_movie_names_with_ratings_above_given_minimum()

  6. Director names who directed at least one movie released in year 2000.

    get_movie_directors_in_given_year()

  7. Actor names who debuted in the 21st century.

    get_actor_names_debuted_in_21st_century()

  8. Director names who directed movies containing "big" as well as movies released in May.

    get_director_names_containing_big_as_well_as_movie_in_may()

  9. Director names who directed movies containing "big" and released in May.

    get_director_names_containing_big_and_movie_in_may()

  10. Reset all rating counts (five-star down to one-star) to 0 for movies released in year 2000.

    reset_ratings_for_movies_in_this_year()

Assignment 3 — IMDb Contd-2

source: django_assignment_004

Focus on aggregation and annotation. Many tasks return Model instances with extra annotated attributes.

  1. Average box_office_collection_in_crores across all movies (round to 3 dp).

    get_average_box_office_collections()

  2. Distinct actor count for each movie → Movie instances with actors_count.

    get_movies_with_distinct_actors_count()

  3. Male & female actor counts per movie → Movie instances with male_actors_count, female_actors_count.

    get_male_and_female_actors_count_for_each_movie()

  4. Distinct role count per movie → Movie instances with roles_count.

    get_roles_count_for_each_movie()

  5. For each role, the number of distinct actors who played it → dict {role: count}.

    get_role_frequency()

  6. Role cast frequency, ordered by most recently released movie first → list of (role, frequency).

    get_role_frequency_in_order()

  7. Per actor: total movies and distinct roles → Actor instances with movies_count, roles_count.

    get_no_of_movies_and_distinct_roles_for_each_actor()

  8. Movies with at least 40 actors → Movie instances with actors_count.

    get_movies_with_atleast_forty_actors()

  9. Average number of actors across all movies (round to 3 dp; 0 if no movies).

    get_average_no_of_actors_for_all_movies()

    get_female_cast_details_from_movies_having_more_than_five_female_cast()

  7. For each actor, return all their movies released in year ≥ 2000 → list of dicts.

Coding Guidelines

  • Write models in models.py; write all task functions in utils.py — both inside the imdb app.
  • Use the exact file, class, function, and field names given in each task.
  • Prefer reverse relations and relation-related fields over multiple manual queries.
  • You may reuse the models from the previous assignment in each continuation.
  • Run all python3 manage.py ... commands from the folder containing manage.py.
