    # Getting started with python dataclasses

    In a software, data is passed between different functions/methods. Did you ever think about the different types of data structures which can be used for this communication?

    When building any software, the choice of data structures used plays a crucial role in easing the development. They should be intuitive, easy to use and should avoid confusion. In the following section, we have described some of the problems you might encounter using different datatypes.

    # Tuples

    With tuples, you need to remember the order of the attributes.

    Suppose you are working with a tuple where the first attribute is Q and the second is Hearts. But, you've assumed that the first attribute of a tuple is Hearts and the second is Q.

    >>> actual_tuple = ('Q', 'Hearts')
>>> queen_of_hearts_tuple = ('Hearts', 'Q')
>>> if queen_of_hearts_tuple[0] ==  actual_tuple[0]:
        print("TRUE")
    else:
        print("FALSE")

FALSE

    On executing the above code, you get FALSE. But, according to your assumption, the result should've been TRUE. When such a mistake happens in a large python project, it's not always easy to debug the code.

    # Dictionaries

    With dictionaries, you must make sure that the 'key names' are used consistently.

    >>> actual_dict = {'rank': 'Q', 'suit': 'Hearts'}
>>> changed_key_name_dict = {'value': 'Q', 'suit': 'Hearts'}
>>> print(changed_key_name_dict['rank'])
KeyError: 'rank'

    If you change a key name in a dictionary in one part of the code, it would raise KeyErrors in other places where you're trying to access the changed key. And remember that these errors are raised in runtime and aren't highlighted by an IDE.

    A better alternative in such scenarios would be the Namedtuple. It has long been used to create readable small data structures.

    # Namedtuple

    Let's look at the above card example using namedtuple.

    >>> from collections import namedtuple
>>> NamedTupleCard = namedtuple('NamedTupleCard', ['rank', 'suit'])

    We defined the NamedTupleCard with rank and suit as attributes. Now, we can simulate the above example using this namedtuple.

    >>> queen_of_hearts = NameTupleCard('Q', 'Hearts')
>>> queen_of_hearts.rank
'Q'
>>> queen_of_hearts
NamedTupleCard(rank='Q', suit='Hearts')
>>> queen_of_hearts == NameTupleCard('Q', 'Hearts')
True

    The namedtuple also comes with some restrictions.

    • It is hard to add default values to some of the fields in a namedtuple.
    • namedtuples are immutable. That is, the values in a namedtuple can't be changed.
    >>> card = NamedTupleCard('7', 'Diamonds')
>>> card.rank = '9'
AttributeError: can't set attribute

    We can overcome these problems by using DataClasses. But, dataclasses can't be used in all scenarios either. They can't replace all the uses of a namedtuple. For instance, if you need your data structure to behave like a tuple, then a named tuple is the best available data structure.

    # DataClasses In Python

    DataClasses are python classes, but are suited for storing data objects.

    • They store data and represent a certain data type.
    • They can be compared to other objects of the same type.
    • DataClass is just a regular class, which means that you can add your own methods to it.

    A DataClass is defined using @dataclass decorator.

    This automatically generates constructor magic method __init__(), string representation method __repr__(), the __eq__() method which overloads == operator and few more for a user-defined class.

    # Example

    Let's create a Position DataClass which hasname, longitude, and lattitude as attributes.

    from dataclasses import dataclass

@dataclass
class Position:
    name: str
    longitude: float
    lattitude: float

    Now, observe the results in the below snippet.

    >>> position = Position('Oslo', 10.8, 59.9)
>>> print(position)
Position(name='Oslo', longitude=10.8, lattitude=59.9)
>>> position.lattitude
59.9
>>> position.longitude
10.8

    Here, we created the Position DataClass object using Oslo as name, 10.8 as longitude, and 59.9 as lattitude. You can observe that the string representation method __repr__() is implemented automatically for you.

    • Try to create multiple objects and compare them using ==.
