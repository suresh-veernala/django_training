      # Clean Code

      # “Any fool can write code that a computer can understand.
      Good programmers write code that humans can understand.”       – Martin Fowler (opens new window)

      By now, you might have known what makes a software good.

      • It should meet the needs of customers.
      • It should be easy to understand and make changes.

      We usually say that a piece of code is clean if it is

      • easy to read & understand, not only to the author but to anyone on the project.
      • conveying the intent of the author.

      A good software should be made of clean code.

      # Clean Code

      Clean code is developed with the reader in mind, and such a software is easy to write, read, and maintain.

      Developers are like authors; great authors are known for writing books that tell a clear, compelling story. They use chapters, headings, and paragraphs to organize their thoughts clearly. Developers work in a very similar fashion, but they use modules, files, classes, and methods instead.

      A professional developer not only solves the given problem(i.e., build a software) but also puts a bit of extra time to focus on the design of the code.

      "Clean code is simple and direct. Clean code reads like well-written prose. Clean code never obscures the designer’s intent but rather is full of crisp abstractions and straightforward lines of control."
      - Grady Booch, author of Object Oriented Analysis and Design with Applications

      "Clean code can be read, and enhanced by a developer other than its original author. It has unit and acceptance tests. It has meaningful names. It provides one way rather than many ways for doing one thing. It has minimal dependencies, which are explicitly defined, and provides a clear and minimal API. Code should be literate since depending on the language, not all necessary information can be expressed clearly in code alone."
      -“Big” Dave Thomas, founder of OTI, godfather of the Eclipse strategy

      # Advantages of Clean Code:

      # Better Use of Your Time

      The first beneficiaries of the clean code are the programmers themselves. If you are working on a project for months, it’s easy to forget things you did in the code, especially when your client comes back with changes. If your code is clumsy, you would need a lot of time to make even a simple change to the software.

      # Easier Debugging

      Whether you write your code dirty or clean, bugs are inevitable. But, clean code will help you to debug faster, regardless of how much experience or expertise you have.

      It’s common for you to ask your colleagues or managers to help you solve the problem. If you’ve written clean code, they can jump in and help you out in no time. Otherwise, it would become a very tough job for them, which they might even refuse to do.

      # Easy to Test

      By writing clean code, automated testing of the code becomes much easier. Large and exhaustive test suites can be built for a clean code base, which brings all the benefits of automated testing (which we have learnt earlier) with it.

      # You’ll Feel Good

      If you’re writing quality, clean code, you will feel super confident. You will not have a fear of fixing defects in your code.

      You'll always welcome changes to the existing features in the application and that means you’re enjoying the software development process.

      # Easier On-boarding of New Team Members

      Using clean code principles helps to get a new programmer onboard within no time. There is no need for him/her to go through lengthy documentation to understand the code; the new programmer can directly jump into it.
