# Resources Reference

A reference of all toolkit resources — what each file is and what it does.


# Skills

Skills are focused implementation guides. The agent loads them based on the task through skill-map.

Skill Description
api-spec-design Standards for writing OpenAPI and Swagger specs, including response structure, error formats, and path design.
backward-compatibility Guidance for making API, schema, and behavior changes without breaking existing consumers.
clean-architecture Maps project structure and layer responsibilities so new code is placed in the right layer.
code-patterns Patterns for multi-step DB work, side effects, state transitions, and other common backend flows.
code-reuse Guidance for reusing existing abstractions and avoiding unnecessary duplication.
code-styling Naming, formatting, imports, constants, and general code-shape conventions.
db-design Reasoning for database design decisions like tables vs fields, relationships, and deletion behavior.
django-orm-queries Database access patterns inside storages, including filtering, annotations, and query efficiency.
explain Framework for explaining code, features, flows, and architecture without changing code.
interactors Structure, validation placement, exception raising, and orchestration patterns for business logic classes.
naming-standards Naming conventions for models, fields, enums, interactors, storages, and API paths.
workflows End-to-end implementation guides for common backend workflows.

Resource Description

django-models Model field definitions, constraints, and model configuration guidance.
django-settings Patterns for reading and defining settings and environment variables.
file-operations-cleanup Local file handling and cleanup patterns for temporary files and file workflows.

debug Systematic process for investigating bugs, errors, and unexpected behavior.
network-calls Outbound HTTP call patterns including retries, timeouts, and exception handling.
poetry Dependency management and environment workflows using Poetry.
presenters Presenter layer patterns, response construction, and response ownership rules.
project-setup One-time local environment setup guidance.
s3-operations Common S3 operations such as uploads, downloads, and presigned URL flows.
sheet-loading Patterns for spreadsheet-loading utility classes and related flows.
test-interactor Guidance for interactor-layer test writing.
test-presenter Guidance for presenter-layer test writing.

# Commands

Commands are the reusable entry points for common workflows.

Command Description
write-tech-prd Converts a requirement into a structured Tech PRD or ordered implementation breakdown. Useful when the task is broad, ambiguous, or needs design analysis.
propose-approaches Researches the codebase and generates 1-2 technically grounded implementation approaches. Preferred when the solution direction is not fixed yet.
implement-with-plan Implements a requirement by following a chosen approach while planning layer-wise changes before editing. Preferred when the direction is already known.
plan-approach Expands a selected approach into an execution-ready plan when more structure is needed before implementation.
build-api Builds or updates API specs for meaningful API changes that need to be communicated clearly.
explain Explains how a feature, flow, or architectural decision works without making changes.
review-impl Reviews completed implementation changes against architecture and code standards.
refactor Improves readability, naming, and structure without changing behavior.
resolve-issue Drives resolution of a concrete issue or tracked problem with implementation focus.
update-docs Updates documentation after code changes, when flows or behavior changed enough to document.
visualize Generates a visual diagram or flow representation for understanding code or architecture.
what-to-test Produces manual testing scenarios for complex or multi-part changes.
write-interactor-tests Adds or updates interactor-layer unit tests.
write-presenter-tests Adds or updates presenter-layer unit tests.
write-storage-tests Adds or updates storage-layer unit tests.
write-view-tests Adds or updates view- and API-layer unit tests.
write-tests Coordinates writing tests across affected layers.

# Rules

Rules are always-on constraints that shape how the agent works across sessions.

Rule Description
agent-behavior Defines core interaction patterns, tool usage, scope control, and when the agent should ask vs act.
skill-map Maps task types to the skills and references the agent should load.

# Hooks

Hooks act as a hard enforcement layer the agent cannot bypass.

Hook Description
hooks.json Maps lifecycle events like preToolUse and beforeShellExecution to hook scripts.
shell-cmds-block Blocks unauthorized or dangerous shell commands.
protected-branch-block Prevents direct writes or pushes to protected branches.
git-cmds-block Blocks destructive git operations like force push, hard reset, and branch deletion.
auto-gen-file-read-block Blocks reads of generated files that should not be manually inspected.
auto-gen-file-write-block Blocks writes to generated files.

# Helpers

Shared utility scripts used by hooks and other toolkit components.

Helper Description
_venv_helper.sh Shared virtual environment discovery helper used by hooks to locate and activate the project venv.

# Review Standards

Contains architecture and code review checklists referenced during review commands. These define what a valid implementation looks like across all layers — storage, interactors, presenters, and API spec.
