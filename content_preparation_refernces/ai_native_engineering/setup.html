# Sync Script Setup

All toolkit resources — skills, commands, rules, hooks, and MCPs — are managed centrally and delivered to your machine via a sync script. The script keeps your local Cursor installation up to date automatically by downloading the latest versions from a central server.

You run it once. After that, everything stays current on its own.


# What Gets Installed

When the sync runs, it populates prompts in your project directories:

Resource Installed To
Commands ~/.cursor/commands/
Rules ~/.cursor/rules/
Skills ~/.cursor/skills/
Agents ~/.cursor/agents/
Hooks ~/.cursor/hooks/ + ~/.cursor/hooks.json
Custom MCPs ~/.cursor/custom-mcps/ + auto-merged into ~/.cursor/mcp.json

Do Not Edit Synced Files Directly

Any changes you make directly to the synced files in these directories will be overwritten the next time the sync runs. If you want custom additions, always create them as separate files alongside the synced ones — do not modify the synced files themselves.


# First-Time Install

Download the sync script and run the install command. This registers the script with your OS scheduler and runs the first sync immediately.

Note: Take access for the s3 bucket nw-agent-toolkit-backend in AWS (from django admin panel or ask your RP).

After access run the below command to download the sync script to your home directory.

export AWS_PROFILE=<aws-profile>

In home directory, run:

aws s3 cp s3://nw-agent-toolkit-backend/client/nw-agent-toolkit-sync.py nw-agent-toolkit-sync.py

Activate any project venv where boto3 is installed and run the file. Removing the venv in that project will later stop the auto-sync

python3 /path/nw-agent-toolkit-sync.py install --profile <aws-profile>

Required:

  • --profile — Your AWS credentials profile name (used to access the resource server). This is saved and reused on every subsequent sync.

Example:

python nw-agent-toolkit-sync.py install --profile nxtwave-dev

The scheduler is registered automatically for your OS — LaunchAgent on macOS, cron on Linux, Task Scheduler on Windows.

Note: The cron trigger is tested on ubuntu & mac

Now update the config.json in .nw-agent-toolkit-backend with the absolute paths of your backend projects as below:

"backend_projects": [
  <Absolute path to repo. eg: /home/nw-tech-prasanth/work/nw_projects/agent-toolkit-backend>
]

and run:

python nw-agent-toolkit-sync.py run --profile [profile name]

# Uninstall

To remove the scheduler (does not delete installed resources):

python nw-agent-toolkit-sync.py uninstall

# After Install

  • Once the first sync completes, open Cursor. All skills, commands, rules, hooks, and MCPs are directly copied to your configured project directories

  • Copy the .pre-commit-config.yaml in .nw-agent-toolkit-backend/cursor/git_hooks and paste in your repos or symlink it. Run the commands 3 & 4 in yaml file doc-strings to install the hooks in your repo.

  • Close & Re-open all the cursor windows for them to sync the prompts.


Keep .cursor Clean Per Project

Do not duplicate the same files both globally in ~/.cursor/ and repo-wise in .cursor/ inside your project. Having the same file in both locations corrupts the context the agent receives and leads to unpredictable behavior.


# Adding Custom Resources

You can still add your own custom rules or resources — either globally in ~/.cursor/ or per-repo in .cursor/ inside your project. The synced resources will work alongside them.

Recommended approach:

  • Start by removing any old or custom rules and running purely on this workflow.
  • If something doesn't behave as per your preference, add a targeted custom rule at that point.
  • If something doesn't work as expected in terms of code patterns or conventions, fill in the feedback sheet so it can be addressed in the central toolkit.
