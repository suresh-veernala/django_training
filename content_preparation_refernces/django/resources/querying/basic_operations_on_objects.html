              # Basic Operations on Objects

              from django.db import models

class Blog(models.Model):
    name = models.CharField(max_length=100)
    tagline = models.TextField()

    def __str__(self):
        return self.name

class Author(models.Model):
    name = models.CharField(max_length=200)
    email = models.EmailField()

    def __str__(self):
        return self.name

class Entry(models.Model):
    blog = models.ForeignKey(Blog, on_delete=models.CASCADE)
    headline = models.CharField(max_length=255)
    body_text = models.TextField()
    pub_date = models.DateField()
    mod_date = models.DateField()
    authors = models.ManyToManyField(Author)
    number_of_comments = models.IntegerField()
    number_of_pingbacks = models.IntegerField()
    rating = models.IntegerField()

    def __str__(self):
        return self.headline

              Note: All examples in this section use the Blog, Author and Entry models defined above.

              To represent database-table data in Python objects, Django uses an intuitive system: A model class represents a database table, and an instance of that class represents a particular record in the database table.

              To create an object, instantiate it using keyword arguments to the model class, then call save() (opens new window) to save it to the database.

              Assuming models live in a file mysite/blog/models.py, here’s an example:

              >>> from blog.models import Blog
>>> b = Blog(name='Beatles Blog', tagline='All the latest Beatles news.')
>>> b.save()

              This performs an INSERT SQL statement behind the scenes. Django doesn’t hit the database until you explicitly call save() (opens new window).

              The save() (opens new window) method has no return value.

              See also, save() (opens new window) takes a number of advanced options not described here. See the documentation for save() (opens new window) for complete details. To create and save an object in a single step, use the create() (opens new window) method.

              # Saving changes to objects

              To save changes to an object that’s already in the database, use save() (opens new window).

              Given a Blog instance b5 that has already been saved to the database, this example changes its name and updates its record in the database:

              >>> b5.name = 'New name'
>>> b5.save()

              This performs an UPDATE SQL statement behind the scenes. Django doesn’t hit the database until you explicitly call save() (opens new window).

              # You can also create an object with create method instead of save

              >>> from blog.models import Blog
>>> Blog.objects.create(name='Beatles Blog', tagline='All the latest Beatles news.')

              # Saving ForeignKey and ManyToManyField fields

              Updating a ForeignKey (opens new window) field works exactly the same way as saving a normal field – assign an object of the right type to the field in question. This example updates the blog attribute of an Entry instance entry, assuming appropriate instances of Entry and Blog are already saved to the database (so we can retrieve them below):

              >>> from blog.models import Blog, Entry
>>> entry = Entry.objects.get(pk=1)
>>> cheese_blog = Blog.objects.get(name="Cheddar Talk")
>>> entry.blog = cheese_blog
>>> entry.save()

              Updating a ManyToManyField (opens new window) works a little differently – use the add() (opens new window) method on the field to add a record to the relation. This example adds the Author instance joe to the entry object:

              >>> from blog.models import Author
>>> joe = Author.objects.create(name="Joe")
>>> entry.authors.add(joe)

              To add multiple records to a ManyToManyField (opens new window) in one go, include multiple arguments in the call to add() (opens new window), like this:

              >>> john = Author.objects.create(name="John")
>>> paul = Author.objects.create(name="Paul")
>>> george = Author.objects.create(name="George")
>>> ringo = Author.objects.create(name="Ringo")
>>> entry.authors.add(john, paul, george, ringo)

              Django will complain if you try to assign or add an object of the wrong type.

              # Retrieving a single object with get()

              If you know there is only one object that matches your query, you can use the get() (opens new window) method on a Manager (opens new window) which returns the object directly:

              >>> one_entry = Entry.objects.get(pk=1)

              # Deleting a single object with get()

              If you know there is only one object that matches your query, you can use the get() (opens new window) method on a Manager (opens new window) which returns the object directly:

              >>> one_entry = Entry.objects.get(pk=1)
>>> one_entry.delete()

              You can use any query expression with get() (opens new window) , just like with filter() (opens new window) - again, see Field lookups (opens new window) .

              Django will complain if more than one item matches the get() (opens new window) query. In this case, it will raise MultipleObjectsReturned (opens new window) , which again is an attribute of the model class itself.
