                          # Social Network Cont'd

                          Social Network Details mentioned in assignment 006

                          There are several users in the network. Each user has

                          1. name - text (max of 100 characters)
                          2. profile_pic - text to store user profile url

                          Every user in the social network should be able to comment on any post. Each comment on application has the following details:

                          1. content - text with which user commented for the post (max of 1000 characters)
                          2. commented_at - date & time when user commented
                          3. commented_by - user who commented to the post
                          4. post - post to which the comment is made.

                          Users can also reply to a comment. These are nothing but 'comments to comments' and
                          the web application should also keep track of

                          1. parent_comment - the comment to which the comment (reply) is made
                          2. Apart from this, it should also have all the details which a typical comment would have (as mentioned above).

                          Users can react to any post/comment. These reactions are limited to the following.

                          • WOW
                          • LIT
                          • LOVE
                          • HAHA
                          • THUMBS-UP
                          • THUMBS-DOWN
                          • ANGRY
                          • SAD

                          A user can react to a post/comment only once.

                          The web application should save the below mentioned details for each of the reactions made.

                          1. post - post on which the user has reacted if this reaction is for a post
                          2. comment - comment on which the user has reacted if this reaction is for a comment
                          3. reaction - the reaction user has made, a text value (max of 100 characters)
                          4. reacted_at - date and time at which the user has reacted
                          5. reacted_by - user who reacted

                          We need to add some additional features to our Social Network Web application, which are described as following.

                          Users can create a group to share posts with the members in that group only.

                          # Group

                          A Group has

                          • name - text (max of 100 characters)
                          • members - All the users who are in this group.

                          Members in a group are unique i.e.; a user cannot have more than one membership in a group.

                          A user can have membership in more than one group. And a group can have more than one member.

                          # Membership

                          Membership in a Group is of two types

                          • Regular
                          • Admin

                          Membership should store the group, member and the type of membership

                          Suggestion

                          Use is_admin boolean to identify user membership in a group

                          All members of a group can create posts in that group.

                          An admin to a group can do additional operations on the group

                          Example:

                          • Adding a new member to the group
                          • Removing a member from the group
                          • Promoting a member as admin

                          A Group can have more than admins, and a user can be an admin for more than one group.

                          # Posts

                          Users should be able to create a post in a group. Each post on the application has the following details:

                          1. content - text with which user creates the post (max of 1000 characters)
                          2. posted_at - date & time when the post is created
                          3. posted_by - the user who has created the post
                          4. group - Group to which the post is associated if the post is created in a group
