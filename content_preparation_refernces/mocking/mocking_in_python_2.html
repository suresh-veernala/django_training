        # Patch

        As we have learnt in the previous section, when a function is expecting an object of another class as an argument (which is a dependency for testing), we can use a Mock object to simulate the behaviour of passed object and isolate the function that is being tested.

        But, how would you isolate the is_weekend_using_get_current_day method described in the below snippet?

        • is_weekend
        def is_weekend_using_get_current_day():
    from get_current_day import get_current_day
    day_as_number = get_current_day()
    return day_as_number in [6, 7]
        • get_current_day
        def get_current_day():
    import datetime
    now = datetime.datetime.now()
    return now.weekday()

        The difference from the previous implementation of 'is_weekend' is that, is_weekend_using_get_current_day doesn't get a calendar object as input anymore. Instead, it imports a function get_current_day.

        Clueless? We use Patch in such scenarios. unittest.mock provides a powerful mechanism for mocking objects, called patch(), which looks up an object in a given module and replaces that object with a Mock.

        We can use Patch as a decorator or a context manager which are explained below.

        # patch as a Decorator

        from unittest.mock import patch
from is_weekend import is_weekend_using_get_current_day

@patch('get_current_day.get_current_day')
def test_is_weekend_using_get_current_day_on_weekend(get_current_day_mock):
    get_current_day_mock.return_value = 6

    result = is_weekend_using_get_current_day()

    assert result is True

        We used patch() as a decorator and passed the target function path to it. The target path is 'get_current_day.get_current_day' which consists of the module name and the function (in this case, the target is get_current_day function). A mock object is created for the get_current_day function and it is passed to the test function as an argument. It can be used for any further modifications like setting return_value or side_effect.

        • Write a test for weekday similar to the above test.

        # patch as a Context Manager

        • Context managers can be used to specify a mock only for a specific part of the code.
        def test_weekend():
    with patch('get_current_day.get_current_day') as get_current_day_mock:
        get_current_day_mock.return_value = 6

        result = is_weekend()

        assert result is True

        Sometimes, you might want to use patch() as a context manager rather than a decorator for the following reasons: - You only want to mock an object for a part of the test scope. - You are already using too many decorators or parameters, which hurts your test’s readability.

        # Patching an Object’s attributes

        • calendar.py
        # my_calendar.py
class MyCalendar:
    
    @staticmethod
    def get_current_day():
        import datetime
        now = datetime.datetime.now()
        return now.isoweekday()
        • is_weekend.py
        # is_weekend.py
def is_weekend_using_calendar_v2():
    from my_calendar import MyCalendar
    calendar_obj = MyCalendar()
    day_as_number = calendar_obj.get_current_day()
    return day_as_number in [6, 7]

        To mock only get_current_day method of MyCalendar class, so that rest of the class behavior works normally, we use patch.object as below.

        from my_calendar import MyCalendar

@patch.object(MyCalendar, 'get_current_day', return_value=6)
def test_weekend(get_current_day):
    from is_weekend import is_weekend

    result = is_weekend()

    assert result is True
        • We use patch.object() when we want to mock only one method of an object/class, instead of the entire object/class.
        • First Argument: The class or object to be mocked
        • Second Argument: The attribute/method to be mocked as a string.
        • Third Argument: mock behavior i.e return_value, side_effect.
