About

The Model Context Protocol (MCP) is an open standard that enables seamless integration between large language model (LLM) applications and external data sources or tools. It is designed to streamline the use of external systems such as databases, APIs, or local files—within AI-powered applications.

MCP is independent of client environments like IDEs or web applications. You can think of MCP as a backend service: clients (such as web apps or development tools) send requests to the MCP server, which then interfaces with the appropriate tools or data sources and returns the results to the client.

Importantly, all tools exposed by MCP servers are included in every AI request, regardless of whether they are relevant to the current task.


To setup mcp servers in cursor follow the reference (opens new window)

Visit the official github repo (opens new window) for the popular mcp servers. There are lot of third party providers also listing these mcp servers.

Note:

  • In cursor whenever you update or add a new mcp server close the existing ide sessions and restart it.

Installation:

  • You can install servers on global level and use them or you can install them in virtual environment but ensure that your virtual environment stays active for mcp servers to run.
  • Installing in virtualenv requires you to install them in every repo.

# Key points

  • For most of the use cases you need not enable all mcp servers. For example, when we want to debug a sentry issue can give access to agent only for sentry server.
  • The tradeoff in enabling all mcp servers by default is all the tools will be sent in every request whether they are relevant to the task or not.
  • When all the tools are sent in the AI request they also consume input tokens and occupy context window.
Best Practices
  • Configure mcp servers with applications that improve our productivity with autonomy.
  • Enable mcp servers based on the task we have. Enabling all the mcp servers will consume context window.
