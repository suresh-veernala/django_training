# About

This section is about how to make the most of Cursor as an AI IDE.

# Key Points

  • Ensure secrets like Api keys or other passwords are not sent to AI and not stored in files which are not in .cursorignore

Effective Usage

  • Use multiple chats to parallelize your work

    You can keep multiple sessions open at once. For example, debugging an issue in one chat while fixing test coverage for a feature in another. This allows you to multitask effectively without context switching or losing ongoing threads of work.

  • Use custom modes tailored to the use case

    Leverage custom modes with specific tool access or behavior suited for your task (e.g., code-only mode, test-focused mode, etc.) to get more relevant and predictable responses.

  • Attach terminal sessions instead of copy-pasting

    Instead of copying logs or command outputs manually into the chat, attach the terminal session directly. This gives the model a real-time view of what’s happening and leads to more accurate debugging.


Use the Right Tool for the Right Task

  • Switch models when something doesn’t work

    Output quality depends on the model being used. If one model struggles, switching to a different model might understand the context or intent better based on its capabilities.

  • Use normal chat mode for brainstorming

    When you’re just exploring an idea or thinking through a problem, the standard chat mode should be sufficient instead of agent mode with unwanted tool capabilities.

  • Reference commits using @git

    You can pull in context from commits directly using @git, which helps when you're working on changes that span multiple files or versions.

  • Use inline chat (Cmd + K) for focused refactors

    Inline chat is ideal for small, file-scoped changes where full project context isn't needed like renaming methods or cleaning up logic in a single file.

  • Guide autocomplete with comments or method names

    Well-named methods and inline comments help the model understand what you intend to do leading to better autocomplete suggestions.

  • Ignore unnecessary files for indexing

    Use .cursorignore to exclude files that aren’t needed for the model’s context, keeping the session clean and reducing noise in responses.


Indexing & Context Management

  • Ensure codebase indexing is successful

    Always confirm that indexing has completed successfully. If you get irrelevant or off-topic responses despite a successful status, try re-triggering the indexing it can help fix stale or broken context.

  • Ignore sensitive files

    Make sure .env files, credential files, and any other sensitive configs are listed in .gitignore or .cursorignore. If they are indexed or exposed in chat, it could lead to security risks and unintended cost implications.

  • Never include credentials in chat context

    Avoid pasting any API keys, AWS credentials, beta env configs, or private tokens directly into the chat. The model doesn’t need them, and it poses security concerns.

  • Exclude temporary or large test files from indexing

    By default, all new files and folders are indexed. If you're generating temporary files during development or testing (like CSVs, logs, local scripts), be sure to ignore them otherwise they can pollute context and skew AI responses.

  • Embedding search can help, but is not perfect

    Cursor’s default code search is useful, but it still depends on keyword relevance and internal scoring. For critical flows, it’s better to explicitly attach or paste context.

  • Use the right keywords to trigger relevant file matches

    Using method names, file names, or domain-specific terms (like UserResource, get_user_profile) in your prompt helps in picking the right files in context.


Using the Agent

  • Let the agent run commands instead of doing it manually

    You don’t need to copy terminal commands and output back into the chat. Let the agent run the commands directly it can access the terminal, execute the command, and use the live output for further reasoning.

  • Use the agent to debug and iterate on test failures

    The agent can run your test suite, understand the failure output, and suggest fixes all within the same workflow. This removes the manual verification of tests passing status.


Improvements in Predefined Prompts
  • Having well-defined, reusable prompts for tasks like interactors, sheet utils, tests, or refactors is helpful but they shouldn’t be treated as static.
  • Over time, as you notice recurring mistakes or quality gaps in outputs, those prompts should evolve.
Capture Repeated Mistakes

If the model keeps making the same kind of mistake across different uses of a predefined prompt like naming inconsistencies, missing validations, or improper structure it's a signal that the prompt itself needs to be improved.

Update the base prompt by:

  • Adding missing context or clear expectations
  • Including examples or constraints if helpful
  • Reflecting any internal conventions that were assumed but never stated

This reduces repeated corrections, improves consistency across devs using the same prompt, and keeps the model aligned with evolving team standards.
