                      # Open API Specification

                      # Writing an OAS

                      For practise you can use Online Editor

                      A typical API consists of the following sections

                      • URL
                      • HTTP Method
                      • Request Data
                      • Response Data
                      • Security
                      • A Unique Operation Id

                      We will be writing OAS in the following sections for the below mentioned API.

                      • URL
                        • http://localhost:8000/api/person/create/
                        • POST METHOD
                      • Request
                        • Body
                      {
    "firstName": "string",
    "lastName": "string",
    "username": "string",
    "age": 24
}
                      • Response
                        • Status Code
                        • Body
                        {
    "createdOn": "string",
    "firstName": "string",
    "lastName": "string",
    "username": "string",
    "age": 25,
    "userId": "string"
}
                      • Security
                      • A Unique Operation Id

                      # Specifying the URL

                      • The API should serve on the following endpoint
                        http://localhost:8000/api/person/create/

                      # Task 1:

                      Speaking of web API, an important information is the root URL which people/programs use to make requests to. This is described by giving a list of accepted schemes (or protocols, like http or https), a host, and a basePath. basePath usually will be your app name.

                      -Update host with localhost:8000 & path with api

                      {
    "schemes": [
        "https",
        "http"
    ],
    "host": "localhost:8000",
    "basePath": "/api/"
}

                      All the apis defined in the api specification file will have https://localhost:8000/api/ as base URL.

                      # Task 2:

                      • Add /person/create/ in path In the paths section we specify the URL tail by adding a new path /person/create/.
                      {
    "paths": {
        "/person/create/": {}
    }
}

                      # Specifying HTTP METHOD

                      The API should allow only POST http method . We can do this by adding method to the path section Lets specify the HTTP Method with which the apis is to be requested.

                      {
    "paths": {
        "/person/create/": {
            "post": {}
        }
    }
}

                      # Specifying Request Format

                      The API should expect the below request data. And shouldn't accept partial request object

                      {
    "firstName": "string",
    "lastName": "string",
    "username": "string",
    "age": 24
}

                      # Specifying Request Body:

                      Lets specify the request body

                      DETAILS 
                      {
    "paths": {
        "/person/create/": {
            "post": {
                "parameters": [
                    {
                        "name": "person",
                        "in": "body",
                        "required": true,
                        "description": "The person to create.",
                        "schema": {
                            "type": "object",
                            "properties": {
                                "firstName": {
                                    "type": "string"
                                },
                                "lastName": {
                                    "type": "string"
                                },
                                "username": {
                                    "type": "string"
                                },
                                "age": {
                                    "type": "integer",
                                    "format": "int64"
                                }
                            }
                        }
                    }
                ]
            }
        }
    }
}

                      # Marking fields as required

                      We can mention the some fields as required i.e when making a request to the API these fields are to be sent or else the server wouldn't accept your request by raises 404 BadRequest . We will be using required field for this purpose

                      Click here for specification 
                      {
    "paths": {
        "/person/create/": {
            "post": {
                "parameters": [
                    {
                        "name": "person",
                        "in": "body",
                        "required": true,
                        "description": "The person to create.",
                        "schema": {
                            "type": "object",
                            "properties": {
                                "firstName": {
                                    "type": "string"
                                },
                                "lastName": {
                                    "type": "string"
                                },
                                "username": {
                                    "type": "string"
                                },
                                "age": {
                                    "type": "integer",
                                    "format": "int64"
                                }
                            },
                            "required": [
                                "firstName",
                                "lastName",
                                "username",
                                "age"
                            ]
                        }
                    }
                ]
            }
        }
    }
}

                      # Specifying Security

                      The API should be accessible only with superuser scope.

                      # Define superuser OAuth scope

                      • securityDefinitions provide a way to specify security related information for APIs.
                      • oauth key in securityDefinitions is used to specify oauth related security definitions.
                      • To add new scopes update the scopes dictionary.
                      Click here for specification 
                      {
    "schemes": [
        "https",
        "http"
    ],
    "host": "localhost:8000",
    "basePath": "/api/",
    "securityDefinitions": {
        "oauth": {
            "tokenUrl": "http://auth.ibtspl.com/oauth2/",
            "flow": "password",
            "scopes": {
                "read": "read users",
                "write": "create users",
                "update": "update users",
                "delete": "delete users",
                "superuser": "super user permission"  # New Scope
            },
            "type": "oauth2"
        }
    }
}

                      # Protect API using OAuth Scopes

                      We can specify the scopes required for an OAuth user to access this api as below.

                      Click here for specification 
                      {
    "paths": {
        "/person/create/": {
            "post": {
                "security": [
                    {
                        "oauth": [
                            "superuser"
                        ]
                    }
                ],
                ...
            }
        }
    }
}

                      # Making APIs readable

                      We can add a short description called summary for our API and also a detailed description about our api in description field. These are used for documentation and help in readability of the API.

                      Click here for specification 
                      {
    "paths": {
        "/person/create/": {
            "post": {
                "summary": "Create New Person",
                "description": "Creates a new person and return the user object. This api is not idempotent.",
                ...
            }
        }
    }
}

                      # Defining Operation Id

                      OperationId is a unique identifier for an API and will be used in different automations.

                      Click here for specification 
                      {
    "paths": {
        "/person/create/": {
            "post": {
                "operationId": "create_person",
                ....
            }
        }
    }
}

                      # Organize Parameters and Responses

                      We can further organize parameters and responses to improve readability & re-usability of the OAS.

                      Click here for specification 
                      {
    ...
    "parameters": {
        "CreateNewPersonParameter": {
            "name": "person",
            "in": "body",
            "description": "The person to create.",
            "schema": {
                "type": "object",
                "properties": {
                    "firstName": {
                        "type": "string"
                    },
                    "lastName": {
                        "type": "string"
                    },
                    "username": {
                        "type": "string"
                    },
                    "age": {
                        "type": "integer",
                        "format": "int64"
                    }
                },
                "required": [
                    "firstName",
                    "lastName",
                    "username",
                    "age"
                ]
            }
        }
    },
    "responses": {
        "CreateNewPersonResponse": {
            "description": "Success Response",
            "schema": {
                "type": "object",
                "properties": {
                    "createdOn": {
                        "type": "string",
                        "format": "datetime"
                    },
                    "firstName": {
                        "type": "string"
                    },
                    "lastName": {
                        "type": "string"
                    },
                    "username": {
                        "type": "string"
                    },
                    "age": {
                        "type": "integer",
                        "format": "int64"
                    },
                    "userId": {
                        "type": "string"
                    }
                },
                "required": [
                    "createdOn",
                    "firstName",
                    "lastName",
                    "username",
                    "age",
                    "userId"
                ]
            }
        }
    },
    "paths": {
        "/person/create/": {
            "post": {
                ...
                "parameters": [
                    {
                        "$ref": "#/parameters/CreateNewPersonParameter"
                    }
                ],
                "responses": {
                    "201": {
                        "$ref": "#/responses/CreateNewPersonResponse"
                    }
                }
            }
        }
    }
}

                      # Optimize using definitions

                      Definitions are a way to reuse any object definitions.

                      Note

                      This doesn't mean that everything should be written in definitions.

                      Click here for specification 
                      {
    "definitions": {
        "User": {
            "type": "object",
            "properties": {
                "firstName": {
                    "type": "string"
                },
                "lastName": {
                    "type": "string"
                },
                "username": {
                    "type": "string"
                },
                "age": {
                    "type": "integer",
                    "format": "int64"
                }
            },
            "required": [
                "firstName",
                "lastName",
                "username",
                "age"
            ]
        },
        "UserWithExtraFields": {
            "allOf": [
                {
                    "$ref": "#/definitions/User"
                },
                {
                    "type": "object",
                    "properties": {
                        "createdOn": {
                            "type": "string",
                            "format": "datetime"
                        },
                        "userId": {
                            "type": "string"
                        }
                    },
                    "required": [
                        "createdOn",
                        "userId"
                    ]
                }
            ]
        }
    },
    "parameters": {
        "CreateNewPersonParameter": {
            "name": "person",
            "in": "body",
            "description": "The person to create.",
            "schema": {
                "$ref": "#/definitions/User"
            }
        }
    },
    "responses": {
        "CreateNewPersonResponse": {
            "description": "Success Response",
            "schema": {
                "$ref": "#/definitions/UserWithExtraFields"
            }
        }
    }
}
