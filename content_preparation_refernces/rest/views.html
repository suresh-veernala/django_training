              # Writing Views with DRF

              Till now, we have learnt how to write serializers. Let us now understand how to build views using a serializer.

              # Scenario

              Suppose, we are building a web application which helps users store and retrieve code snippets. A code snippet has a title and code - both of which are strings.

              Our web application should provide the below two APIs.

              1. An API to create a Snippet in DB.
              2. An API to retrieve all the Snippets from the DB.

              To build any typical web API, a backend developer should be able to clearly answer the following questions.

              • What information do you get in the API call? (Request)
                • HTTP Method
                • URL
                • Request Data
              • How should you handle the Request? (API Logic)
                • The business logic or Python code
                • Required DB operations (ORM queries)
              • What are you expected to return? (Response)
                • HTTP Status Code
                • Response Data

              Answers to the above mentioned questions for the "create snippet" API are:

              • Request:

                • We want the API to be served on POST method
                • URL to be served - 'http://localhost:8000/snippets/'
                • Request Body
                {
    "title": "New Code Snipppet",
    "code": "def some_function(....."
}

              • API Logic:

                • The api should create a record with the given data (title, code) in the Database.

              • Response:

                • The api should return an HTTP Response with the status code 200. It should also include following data in the Response body.
                {
    "id": 1,
    "title": "New Code Snipppet",
    "code": "def some_function(....."
}

              After listing all the necessary details required to implement an API, we can write an API in five steps.

              1. Create a dummy view and link to the URL
              2. Handle Request data
              3. Create a Dummy Response
              4. Integrate the API logic
              5. Handle Invalid Request data

              # Handling request data

              Our API is properly integrated with urls. Let's write serializers to validate and convert the given request data to a python object.

              from rest_framework import serializers
from rest_framework.decorators import api_view
from rest_framework.response import Response


class CreateSnippetRequest:
    def __init__(self, code, title=''):
        self.title = title
        self.code = code


class CreateSnippetRequestSerializer(serializers.Serializer):
    title = serializers.CharField(required=False, allow_blank=True,
                                  max_length=100)
    code = serializers.CharField()

    def create(self, validated_data):
        return CreateSnippetRequest(**validated_data)


@api_view(['POST'])
def create_snippet(request):
    serializer = CreateSnippetRequestSerializer(data=request.data)
    if serializer.is_valid():
        request_obj = serializer.save()
        print(
            "title: {}, code: {}".format(request_obj.title, request_obj.code)
        )
        return Response()

class CreateSnippetRequestSerializer(serializers.Serializer):

                                  max_length=100)
    code = serializers.CharField()

    def create(self, validated_data):
        return CreateSnippetRequest(**validated_data)


class CreateSnippetResponseClass:
    def __init__(self, id, title, code):
        self.id = id
        self.title = title
        self.code = code


class CreateSnippetResponseSerializer(serializers.Serializer):
    id = serializers.IntegerField()
    title = serializers.CharField(required=False, allow_blank=True,
                                  max_length=100)
    code = serializers.CharField()

    def create(self, validated_data):

def create_snippet_in_db(title, code):
    """
    Create and return a new `Snippet` instance, given the validated data.
    """
    from snippets.models import Snippet

@api_view(['POST'])
def create_snippet(request):
    serializer = CreateSnippetRequestSerializer(data=request.data)
    if serializer.is_valid():
        request_obj = serializer.save()
        new_snippet_obj = \
            create_snippet_in_db(request_obj.title, request_obj.code)

        response_serializer = CreateSnippetResponseSerializer(new_snippet_obj)
        return Response(response_serializer.data)

              • In case of invalid request data, we have to send an HTTP response with 400 as the status code to indicate a BadRequest.
              # Create your views here.


from rest_framework import serializers
from rest_framework.decorators import api_view
from rest_framework.response import Response


class CreateSnippetRequest:
   def __init__(self, code, title=''):
       self.title = title
       self.code = code


class CreateSnippetRequestSerializer(serializers.Serializer):
   title = serializers.CharField(required=False, allow_blank=True,
                                 max_length=100)
   code = serializers.CharField()

   def create(self, validated_data):
       return CreateSnippetRequest(**validated_data)


class CreateSnippetResponseClass:
   def __init__(self, id, title, code):
       self.id = id
       self.title = title
       self.code = code


class CreateSnippetResponseSerializer(serializers.Serializer):
   id = serializers.IntegerField()
   title = serializers.CharField(required=False, allow_blank=True,
                                 max_length=100)
   code = serializers.CharField()

   def create(self, validated_data):
       return CreateSnippetResponseClass(**validated_data)


def create_snippet_in_db(title, code):
   """
   Create and return a new `Snippet` instance, given the validated data.
   """
   from snippets.models import Snippet
   return Snippet.objects.create(title=title, code=code)


@api_view(['POST'])
def create_snippet(request):
   serializer = CreateSnippetRequestSerializer(data=request.data)
   is_invalid_request_data = not serializer.is_valid()

   if is_invalid_request_data:
       return Response(status=400)

   request_obj = serializer.save()

   new_snippet_obj = create_snippet_in_db(
       title=request_obj.title,
       code=request_obj.code
   )

   response_serializer = CreateSnippetResponseSerializer(new_snippet_obj)
   return Response(response_serializer.data)
