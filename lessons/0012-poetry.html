Why Poetry?

Single config file

One pyproject.toml replaces setup.py, setup.cfg, and multiple requirements*.txt files.

Dependency locking

poetry.lock pins every package to an exact version — identical installs on every machine and in CI.

Intelligent resolution

Full SAT-solver resolves transitive dependency conflicts that pip silently ignores.

Safe removes

Won't uninstall a package still needed by another — pip uninstall has no such check.

Dev / prod groups

--group dev keeps test tools out of production. poetry install --only main for prod deploys.

pip compatible

Works with all existing PyPI packages. Use poetry export to generate requirements.txt when needed.

pip + requirements.txt (old)

  • Manually run pip freeze and commit output
  • No lock — teammate installs a newer sub-dep and things break
  • Multiple files: base.txt, dev.txt, prod.txt
  • No separation of dev-only vs prod dependencies
  • Version conflicts silently overwrite packages

Poetry (modern)

  • One command adds + installs + locks a dependency
  • poetry.lock pins every package to exact version + hash
  • One pyproject.toml with named groups
  • --group dev for test tools, linters, etc.
  • Resolver detects conflicts before install

Understanding pyproject.toml

[tool.poetry] name = "imdb" # project name version = "0.1.0" authors = ["You <you@example.com>"] [tool.poetry.dependencies] # prod deps — installed always python = "^3.11" # ^ = ≥3.11, <4.0 django = "^5.0" # ^ = ≥5.0, <6.0 djangorestframework = "^3.15" [tool.poetry.group.dev.dependencies] # dev-only — skipped in prod pytest = "*" pytest-django = "*" factory-boy = "*" [[tool.poetry.source]] # private registry (CodeArtifact) name = "codeartifact" url = "https://<domain>-<account>.d.codeartifact.<region>.amazonaws.com/pypi/<repo>/simple/" priority = "supplemental" [build-system] requires = ["poetry-core"] build-backend = "poetry.core.masonry.api"

The caret (^) means "compatible with this major version". A bare * means "latest available".

Part 1

Install Poetry

Source: DigitalOcean — Installing Poetry

Important
Do not install Poetry via pip install poetry. Always use the official installer below.
  • Download and run the official installer 
    curl -sSL https://install.python-poetry.org | python3 -
  • Add Poetry to your PATH — append to ~/.zshrc (macOS) or ~/.bashrc (Linux) 
    export PATH="$HOME/.local/bin:$PATH"
    Then reload: source ~/.zshrc
  • Verify 
    poetry --version
    Expected output
    Poetry (version 1.x.x)
Part 2

Install AWS CLI v2

Source: AWS Docs — Installing the AWS CLI

  • Download the installer 
    curl "https://awscli.amazonaws.com/AWSCLIV2.pkg" -o "AWSCLIV2.pkg"
    Or download in your browser: AWSCLIV2.pkg
  • Run the installer 
    sudo installer -pkg AWSCLIV2.pkg -target /
    (Or double-click the .pkg file.)
  • Verify 
    which aws && aws --version
    Expected output
    /usr/local/bin/aws
    aws-cli/2.x.x Python/3.x.x Darwin/...
  • Download and extract 
    curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
  • Install
    sudo ./aws/install
  • Verify 
    aws --version
    Expected output
    aws-cli/2.x.x Python/3.x.x Linux/...
  • Download the MSI installer
    AWSCLIV2.msi
  • Run the installer — double-click and follow prompts.
  • Verify (in a new Command Prompt) 
    aws --version
    Expected output
    aws-cli/2.x.x Python/3.x.x Windows/...

Configure AWS credentials

  • Run the configuration wizard 
    aws configure
    Enter your Access Key ID, Secret Access Key, default region, and output format (json).
  • Verify credentials work 
    aws sts get-caller-identity
    Expected output
    JSON with your UserId, Account, and Arn.
Part 3

Connect Poetry to AWS CodeArtifact

Uses the poetry-codeartifact-login plugin — handles token fetching automatically.

  • Install the plugin (once per machine) 
    poetry self add poetry-codeartifact-login
  • Add your CodeArtifact source to pyproject.toml

    URL must end with /simple/ — without it Poetry cannot find packages.

    [[tool.poetry.source]]
name     = "codeartifact"
url      = "https://<domain>-<account-id>.d.codeartifact.<region>.amazonaws.com/pypi/<repo>/simple/"
priority = "supplemental"
  • Log in using your AWS profile 
    poetry aws-login codeartifact --profile <aws-profile-name>
    Using the default profile: poetry aws-login codeartifact
    This fetches a 12-hour token and stores it in Poetry's credential store automatically.
  • Verify — install dependencies 
    poetry install
Token expiry
CodeArtifact tokens last 12 hours. Re-run poetry aws-login codeartifact the next day if you see authentication errors.
Part 4

Essential Commands

Install & manage packages

CommandWhat it does
poetry installCreate virtualenv + install all deps from poetry.lock
poetry install --no-rootInstall deps only, skip the project package itself
poetry install --only mainInstall only production deps (skip dev group)
poetry add djangoAdd a production dependency and update the lock file
poetry add "django==5.0.3"Add a pinned (exact version) dependency
poetry add --group dev pytestAdd a dev-only dependency
poetry remove requestsRemove a dependency (checks transitive deps first)
poetry showList all installed packages
poetry show --treeShow full dependency tree

Lock file

CommandWhat it does
poetry lockResolve and update poetry.lock (upgrades allowed)
poetry lock --no-updateRe-generate lock file without upgrading any package
poetry updateUpgrade all packages to latest allowed and update lock
poetry update djangoUpgrade only django and update lock
Rule
Always commit poetry.lock. Never commit the .venv/ folder.

Project-level venv (manually created)

  • Create the venv inside your project folder 
    python -m venv .venv
  • Tell Poetry to use it 
    poetry env use .venv/bin/python
  • (Optional) Set globally — Poetry always uses .venv/ in-project 
    poetry config virtualenvs.in-project true
  • Verify 
    poetry env info
    Expected output
    Path: .../.venv  ·  Python: 3.x.x  ·  Valid: True

Migrating an Existing Django Project

  • Initialise Poetry in the project root 
    poetry init
    Answer the wizard prompts. Skip adding dependencies interactively.
  • Import from requirements.txt 
    cat requirements.txt | grep -v "^#" | xargs poetry add
  • Move dev-only deps to the dev group 
    poetry add --group dev pytest pytest-django factory-boy
  • Export back to requirements.txt if needed 
    poetry export -f requirements.txt --output requirements.txt --without-hashes
