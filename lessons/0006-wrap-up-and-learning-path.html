Part 3 — What To Do In Each Block

10 min · Environment Setup

Goal: a running Django dev server before anything else.

  1. Confirm Python: python3 --version
  2. Create a virtual environment: python3 -m venv venv
  3. Activate it: source venv/bin/activate (macOS/Linux)
  4. Install dependencies: pip install -r requirements.txt
  5. Verify Django: python -m django --version
  6. Run the server: python3 manage.py runserver and open the local URL
If you hit a host error
Add "*" to ALLOWED_HOSTS in settings.py for local development, and always run manage.py commands from the folder that contains it.

40 min · Django Tutorial – Part 1

Goal: build a project + app end to end and serve your first response. Focus areas:

  • Create a project — django-admin startproject (see Module 2).
  • Create an app — python3 manage.py startapp, then register it in INSTALLED_APPS.
  • URL routing — wire the root urls.py to the app's urls.py via include().
  • Views — write a view that returns a response.
  • Run the application — confirm the route works in the browser.

Primary source: Writing your first Django app, Part 1.

1 hr 30 min · IMDb Assignments (core)

Goal: model the IMDb domain and perform CRUD + basic querying with the ORM. Work in order:

  1. Design the models — Movie, Actor, Director, Cast, Rating. See the IMDb Data Model.
  2. Make & apply migrations — makemigrations → migrate.
  3. Populate the database — implement populate_database(...).
  4. Basic ORM tasks — work through Assignment 1 tasks (counts, filters, updates, deletes).
Outcome for this block
A working IMDb schema with data loaded, and Assignment 1 functions passing. This is the foundation the API block builds on.

50 min · Build REST APIs

Goal: connect your IMDb models to HTTP endpoints that return JSON (see Module 4). Implement 2–3 endpoints:

  1. GET /movies/<movie_id>/ — return complete movie details.
  2. GET /movies/<movie_id>/actors/ — return all actors in the movie (via the Cast relation).
  3. POST /movies/ — create a new movie from a JSON body.

For each: write the URL, the view, query with the ORM, and return a JSON response (plain Django or DRF — both are valid, see Module 4).

30 min · OpenAPI Specification

Goal: document the APIs you just built and view them in Swagger UI (see Module 5).

  1. Describe the document basics: info, host/basePath, schemes.
  2. Define each endpoint under paths with parameters and responses.
  3. Define reusable schemas in definitions and reference them with $ref.
  4. Render the spec in Swagger UI and verify the examples.

Advanced IMDb Assignments

Goal: deepen ORM skill with filtering, aggregation, and optimization. Continue the IMDb track:

  • IMDb Contd-1 — field lookups, date filters, complex conditions. Tasks →
  • IMDb Contd-2 — aggregation & annotation (counts per movie/actor/role). Tasks →
  • IMDb Contd-3 — efficient population, bulk ops, related-object queries. Tasks →
Explain with SQL
For optimization tasks, inspect the generated SQL and reason about "order of N" — why select_related / prefetch_related reduce query counts.
