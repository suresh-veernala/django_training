Part 3 — What To Do In Each Block

20 min · Environment Setup

Goal: a clean Python environment with only the basic packages needed for this training.

  1. Create a practice folder: mkdir imdb && cd imdb
  2. Confirm Python is available: python3 --version
  3. Create a virtual environment: python3 -m venv .venv
  4. Activate it: source .venv/bin/activate (macOS/Linux)
  5. Upgrade the installer: python -m pip install --upgrade pip
  6. Install the basic packages: python -m pip install "Django>=5.1,<5.2" djangorestframework
  7. Save the package list: python -m pip freeze > requirements.txt
  8. Verify Django: python -m django --version
Keep the install small
Start with Django and djangorestframework. Do not add database drivers, Swagger generators, or deployment packages until a task actually needs them.

35 min · Django Tutorial – Part 1

Goal: create the project before running any manage.py commands.

  1. Create the project — django-admin startproject imdb . (the dot keeps manage.py in the current folder).
  2. Check the project — confirm you now have manage.py and an imdb/ folder.
  3. Create the app — python manage.py startapp movies.
  4. Register apps — add rest_framework and movies to INSTALLED_APPS.
  5. Write one view — return a simple response from movies/views.py.
  6. Wire URLs — create movies/urls.py, then include it from imdb/urls.py.
  7. Initialize the database — python manage.py migrate.
  8. Run the server — python manage.py runserver and open the local URL.

Primary source: Writing your first Django app, Part 1.

If a command fails
Check two things first: the virtual environment is active, and you are in the folder that contains manage.py.

1 hr 15 min · IMDb Assignment 1

Goal: model the IMDb domain and perform CRUD + basic querying with the ORM. Work in order:

  1. Design the models — Movie, Actor, Director, Cast, Rating. See the IMDb Data Model.
  2. Make & apply migrations — makemigrations → migrate.
  3. Populate the database — implement populate_database(...).
  4. Basic ORM tasks — work through Assignment 1 tasks (counts, filters, updates, deletes).
Outcome for this block
A working IMDb schema with data loaded, and Assignment 1 functions passing. This is the foundation the API block builds on.

40 min · IMDb Query Practice

Goal: strengthen ORM fluency before using the queries inside API views. Continue in this order:

  1. IMDb Contd-1 — filtering, field lookups, date queries, and complex conditions. Tasks →
  2. Pick 2 aggregation tasks from IMDb Contd-2 if time allows. Tasks →
  3. Inspect one query — print or log the generated SQL for a query you wrote.
Explain with SQL
When a query feels confusing, inspect the generated SQL and explain it in plain English before moving on.

40 min · Build REST APIs

Goal: connect your IMDb models to HTTP endpoints that return JSON (see Module 4). Implement 2–3 endpoints:

  1. GET /movies/<movie_id>/ — return complete movie details.
  2. GET /movies/<movie_id>/actors/ — return all actors in the movie (via the Cast relation).
  3. POST /movies/ — create a new movie from a JSON body.

For each: write the URL, the view, query with the ORM, and return a JSON response (plain Django or DRF — both are valid, see Module 4).

30 min · OpenAPI Specification

Goal: document the APIs you just built and view them in Swagger UI (see Module 5).

  1. Describe the document basics: info, host/basePath, schemes.
  2. Define each endpoint under paths with parameters and responses.
  3. Define reusable schemas in definitions and reference them with $ref.
  4. Render the spec in Swagger UI and verify the examples.

Stretch After The 4 Hours

Goal: deepen ORM skill with aggregation and optimization after the main hands-on path is complete:

  • IMDb Contd-2 — aggregation & annotation (counts per movie/actor/role). Tasks →
  • IMDb Contd-3 — efficient population, bulk ops, related-object queries. Tasks →

Part 4 — Expected Outcome

By the end of the hands-on you should have:

  • ✅ Completed Django Tutorial – Part 1
  • ✅ Modelled the IMDb schema and completed Assignment 1
  • ✅ Practiced IMDb filtering/lookups before using them in views
  • ✅ Built 2–3 REST APIs on the IMDb models
  • ✅ Written an OpenAPI spec and rendered it in Swagger UI
  • ✅ Know exactly where to continue: IMDb Contd-2/3
