# Mission: Django Backend Development

## Why
Become productive on a production-style Django backend codebase — able to model data, query it with the ORM, build REST APIs, and document them with OpenAPI. This mirrors a 2-hour instructor session + 4-hour hands-on internal training, ending with real assignments (IMDb, Social Network).

## Success looks like
- Explain Django's MVT architecture and trace a request from URL to JSON response
- Navigate a Django project structure (`manage.py`, `settings.py`, `urls.py`, apps, migrations)
- Design models and relationships (One-to-One, ForeignKey, Many-to-Many)
- Perform CRUD with the ORM: `create / get / filter / all / update / delete / order_by / count / exists`
- Build REST APIs (GET/POST/PUT/DELETE) with both plain Django `HttpResponse` and DRF
- Write an OpenAPI 3.0 spec and render it in Swagger UI
- Continue independently via the internal learning doc + assignments

## Constraints
- Sessions are short (10–40 mins each) — lessons must be completable quickly
- Backend / API focus — server-side HTML templating is out of scope
- The production codebase mixes plain Django responses and DRF — both must be understood

## Out of scope (for now)
- Django templates / server-side HTML rendering
- Deep async, caching, deployment topics
- Frontend frameworks
